# TerminUS 用户体验设计 (UX Design)

## 1. 核心目标 (Design Goals)

- **极简主义**: 所有交互发生在终端内，无图形 UI。
- **高性能交互**: 零延迟的鼠标、键盘反馈。
- **现代化 TUI**: 利用现代终端支持（如 24-bit TrueColor、SGR-mouse、私有 Unicode 图标集）创造富有质感的界面。

## 2. 交互逻辑 (Interaction Model)

TerminUS 虽是终端系统，但深度集成鼠标操作：

- **全局菜单栏**: 顶部一行作为系统状态与全局菜单入口（Ctrl+Space 激活）。
- **多窗口管理 (TUI Windows)**: 通过呼叫虚拟机软件直接在物理机上建立新的wgpu的窗口，或一个由自己输出内容接受交互的新终端窗口管理 TEP 应用。
- **鼠标热点 (Mouse Hotspots)**:
  - 鼠标点击：激活输入框或按钮。
  - 鼠标拖拽：调整面板大小或滚动内容。
  - 鼠标右键：弹出上下文菜单。

## 3. UI 视觉规范 (Visual Guidelines)

- **配色方案**: 默认采用马卡龙色配色，不支持完全的主题自定义，一切为开发者个人审美服务并要求用户接受（系统开发的苹果模式）。
- **排版**: 统一采用半角字符绘制 UI 框架，利用特定图标字符（如 Nerd Fonts）作为交互标识。
- **反馈**: 引入轻量级的闪烁或色彩变化来标识活动区域。

## 4. 与宿主机 CLI 的结合 (Host Integration)

- **直通命令**: 用户在物理机执行 `vm ls /home` 时，TerminUS 内核能无感知地启动并返回结果。
- **剪贴板共享**: 实现宿主机剪贴板与虚拟系统 TUI 剪贴板的双向同步。
- **文件直挂**: 支持将宿主机物理文件夹临时挂载到虚拟系统的 `/mnt/host` 目录。
